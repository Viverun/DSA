### What Is a While Loop, Conceptually

A while loop repeats a block of code **as long as a condition stays true**. Unlike a for loop — which is built around iterating through a known sequence or a fixed count — a while loop doesn't know in advance how many times it will run. It just keeps checking a condition before every pass, and stops the moment that condition becomes false.

This makes while loops the right tool whenever the number of repetitions **depends on something happening during the loop itself**, rather than being decided upfront.

---

### Basic Syntax

```python
n = 5

while n > 0:
    print(n)
    n -= 1
```

**Output:**

```
5
4
3
2
1
```

The structure, piece by piece:

- `while` keyword, followed by a condition
- a colon (`:`) — same requirement as `if` and `for`
- an indented block below it, which runs repeatedly

**Crucially:** the condition is checked **before** each run of the loop body, including the very first one. If the condition is false right from the start, the loop body never executes at all — not even once.

```python
n = 0

while n > 0:
    print("This will never print.")

print("Loop skipped entirely.")
```

**Output:**

```
Loop skipped entirely.
```

### Worked Example: Factorial

The factorial of `n` is the product of every positive integer from 1 up to `n`. This is a textbook while-loop use case — you don't know upfront "how many times" in the for-loop sense; instead, you keep multiplying and decrementing until you naturally hit the stopping point.

```python
n = 5
factorial = 1

while n > 0:
    factorial *= n   # multiply running total by current n
    n -= 1            # decrement n toward the stopping condition

print(f"Factorial of 5 is: {factorial}")
```

**Output:**

```
Factorial of 5 is: 120
```

Trace through it manually once so the mechanics are fully clear:

|Iteration|`n` before|`factorial *= n`|`n` after|
|---|---|---|---|
|1|5|1 × 5 = 5|4|
|2|4|5 × 4 = 20|3|
|3|3|20 × 3 = 60|2|
|4|2|60 × 2 = 120|1|
|5|1|120 × 1 = 120|0|

Loop stops here since `n > 0` is now false (`n` is 0). Final answer: `120`.

Note `factorial *= n` and `n -= 1` — these are Python's compound assignment operators, shorthand for `factorial = factorial * n` and `n = n - 1`. You'll use these constantly; they read cleaner and are the idiomatic way to update a variable based on its own current value.

### For vs While — When to Use Which

This is a genuinely important design decision you'll make constantly going forward, so it's worth stating clearly:

|Use a **for loop** when...|Use a **while loop** when...|
|---|---|
|You know the number of iterations in advance|The number of iterations depends on runtime conditions|
|You're iterating over an existing sequence (list, string, range)|You're repeating "until something happens" (a value found, a condition met, user stops)|
|Example: process every element in an array|Example: keep halving a number until it drops below 1|
|Example: print numbers 1 through 100|Example: keep asking for input until it's valid|

If you find yourself writing a for loop and awkwardly trying to `break` out of it based on some unrelated condition, that's often a sign a while loop is the more natural fit — and vice versa.

---

### The Risk of Infinite Loops

Because a while loop's stopping point isn't automatic the way `range()` guarantees an eventual end, it's entirely possible to write a while loop that **never stops** if you forget to update the variable the condition depends on:

```python
n = 5

while n > 0:
    print(n)
    # forgot to decrement n — this loop never ends!
```

Running this would print `5` forever until you manually interrupt the program. **Every while loop needs something inside its body that eventually makes the condition false.** Before moving on from any while loop you write, double-check: is there a line that actually changes the variable being tested?

---

### The `while True` + `break` Pattern (Deliberate Infinite Loop)

Sometimes you _want_ to loop indefinitely, and exit only when a specific event happens inside the loop — rather than checking a simple condition up front. The idiomatic way to write this is `while True:` combined with a `break` statement:

```python
count = 0

while True:
    count += 1
    print(f"Count is {count}")
    if count == 5:
        break
```

**Output:**

```
Count is 1
Count is 2
Count is 3
Count is 4
Count is 5
```

`break` immediately exits the loop the moment it's hit, regardless of the `while True` condition being permanently true. This pattern is common when the exit condition is more naturally checked _partway through_ the loop body rather than neatly at the top — you'll see this constantly once you're processing input streams or searching for a stopping event mid-computation.

### Python Has No `do-while` — Here's How to Simulate One

Some languages offer a `do-while` loop, which guarantees the loop body runs **at least once** before the condition is even checked (the opposite ordering from a regular while loop). Python doesn't have a dedicated `do-while` keyword — but you rarely need one, because `while True` + `break` naturally covers the same need:

```python
n = 0

while True:
    print(f"This runs at least once. n = {n}")
    n += 1
    if n >= 3:
        break
```

**Output:**

```
This runs at least once. n = 0
This runs at least once. n = 1
This runs at least once. n = 2
```

The key structural difference from a normal while loop: the condition check (`if n >= 3: break`) is placed at the **end** of the loop body instead of at the top — which is exactly what "check after running once" means. This comes up in a real DSA context most often with input-validation-style logic, covered next.

### Common Real-World Use Case: Input Validation

A very natural while-loop scenario: keep asking the user for input until they provide something valid. This genuinely can't be done cleanly with a for loop, since you have no idea in advance how many attempts it'll take.

```python
number = int(input("Enter a positive number: "))

while number <= 0:
    print("That's not positive. Try again.")
    number = int(input("Enter a positive number: "))

print(f"Thanks! You entered {number}.")
```

**Input (across attempts):**

```
-5
0
7
```

**Output:**

```
Enter a positive number: That's not positive. Try again.
Enter a positive number: That's not positive. Try again.
Enter a positive number: Thanks! You entered 7.
```

## Cheat Sheet — 1.6 While Loops

|Task|Code|
|---|---|
|Basic while loop|`while condition:`|
|Compound update (shorthand)|`x -= 1`, `x += 1`, `x *= 2`, etc.|
|Deliberate infinite loop|`while True:`|
|Exit a loop early|`break`|
|do-while equivalent (run at least once)|`while True:` with the condition check + `break` at the **end** of the body|

## Common Pitfalls to Avoid

- **Forgetting to update the loop variable**, creating an accidental infinite loop. Always check: does something inside the body move the condition toward becoming false?
- **Assuming the loop body runs at least once.** A regular `while` checks its condition _before_ the first run — if it starts false, the body never executes, unlike a `do-while` in other languages.
- **Using `while True` without a `break` anywhere in the body.** This is always a genuine infinite loop and will hang your program.
- **Choosing a while loop when a for loop fits better (or vice versa).** If the number of iterations is genuinely known ahead of time, a for loop is usually cleaner and less error-prone — save while loops for condition-driven repetition.
