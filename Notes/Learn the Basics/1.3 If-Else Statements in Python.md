### Why Conditionals Matter

Conditional statements let your program make decisions — run one block of code if something is true, a different block if it's false. This is the first real building block of "logic" in programming, and nearly every algorithm you write from here on will lean on it constantly: base cases in recursion, branch conditions in searches, edge-case handling, comparisons in sorting — all of it sits on top of if-else.

---

### 1. The Basic `if` Statement

```python
age = 20

if age >= 18:
    print("You are an adult.")
```

**Output:**

```
You are an adult.
```

The structure:

- `if` keyword, followed by a condition
- a **colon (`:`)** at the end of the condition line — this is mandatory
- the code to run sits on the **next line, indented**

There are no curly braces `{ }` in Python. **Indentation itself defines the block.** This is not a style choice — it's how Python knows what belongs inside the `if` and what doesn't. Standard indentation is 4 spaces. Get consistent with this immediately; inconsistent indentation is a syntax error in Python, not just messy formatting.

---

### 2. Adding `else`

```python
age = 15

if age >= 18:
    print("You are an adult.")
else:
    print("You are not an adult.")
```

**Output:**

```
You are not an adult.
```

`else` needs no condition of its own — it's the catch-all for "if the above wasn't true." Note it also ends in a colon, and its block is indented the same way.

---

### 3. Taking Input and Branching on It

Putting 1.1's input-handling together with a conditional — this exact pattern (read input, convert type, branch) is something you'll write dozens of times:

```python
age = int(input("Enter your age: "))

if age >= 18:
    print("You are an adult.")
else:
    print("You are not an adult.")
```

**Input:**

```
20
```

**Output:**

```
Enter your age: You are an adult.
```

Note: `input("Enter your age: ")` — you can pass a string directly into `input()` as a prompt, and it will be displayed before waiting for the user to type. This saves you a separate `print()` call.

### 4. Multiple Conditions with `elif`

When you have more than two branches, Python uses `elif` — short for "else if," fused into a single keyword. (There is no separate `else if` in Python; it's always `elif`, one word.)

```python
marks = 54

if marks < 25:
    print("Grade: F")
elif marks >= 25 and marks <= 44:
    print("Grade: E")
elif marks >= 45 and marks <= 49:
    print("Grade: D")
elif marks >= 50 and marks <= 59:
    print("Grade: C")
elif marks >= 60 and marks <= 69:
    print("Grade: B")
elif marks >= 70:
    print("Grade: A")
else:
    print("Invalid marks entered.")
```

**Output:**

```
Grade: C
```

**How the flow works:** Python checks conditions top to bottom and runs the **first** one that's true, then skips everything else in the chain — even if a later condition would also technically be true. Order matters.

---

### 5. Logical Operators: `and`, `or`, `not`

Python spells out its logical operators as actual words instead of symbols:

|Operator|Meaning|
|---|---|
|`and`|both conditions must be true|
|`or`|at least one condition must be true|
|`not`|inverts a condition (true becomes false, and vice versa)|

```python
age = 25
has_id = True

if age >= 18 and has_id:
    print("Entry allowed.")

if age < 18 or not has_id:
    print("Entry denied.")
```

**Output:**

```
Entry allowed.
```

---

### 6. Chained Comparisons — A Genuine Python Convenience

Here's a distinctly Python feature worth highlighting: you can chain comparisons directly, without repeating the variable and without `and`.

Instead of:

```python
if marks >= 25 and marks <= 44:
```

You can write:

```python
if 25 <= marks <= 44:
```

Both do exactly the same thing — Python evaluates it as "is 25 less than or equal to marks, **and** is marks less than or equal to 44" — but the chained version reads more naturally and is less error-prone to type out, especially when you have several range checks stacked in a row. Use it freely.

```python
marks = 54

if marks < 25:
    print("Grade: F")
elif 25 <= marks <= 44:
    print("Grade: E")
elif 45 <= marks <= 49:
    print("Grade: D")
elif 50 <= marks <= 59:
    print("Grade: C")
elif 60 <= marks <= 69:
    print("Grade: B")
elif marks >= 70:
    print("Grade: A")
else:
    print("Invalid marks entered.")
```

---

### 7. Simplifying the Ladder — Dropping Redundant Lower Bounds

Look closely at the grading logic above: by the time execution reaches the `elif 25 <= marks <= 44` line, Python has _already_ confirmed `marks` is not less than 25 (otherwise it would have stopped at the first `if`). That means the lower bound `25 <=` is redundant information — it's guaranteed by simply having reached that line at all.

You can simplify the entire ladder to check **only the upper bound** at each step:

```python
marks = 54

if marks < 25:
    print("Grade: F")
elif marks <= 44:
    print("Grade: E")
elif marks <= 49:
    print("Grade: D")
elif marks <= 59:
    print("Grade: C")
elif marks <= 69:
    print("Grade: B")
else:
    print("Grade: A")
```

**Output:**

```
Grade: C
```

This is functionally identical to the longer version but noticeably cleaner — fewer comparisons, fewer chances to make a typo in a range boundary, and easier to update later if grade cutoffs change (you only touch one number per line instead of two). This kind of simplification — recognizing what's _already guaranteed_ by the flow of control — is a habit worth building early; it comes up constantly once you start writing more involved conditional logic.

### 8. Nested `if` Statements

Conditionals can sit inside other conditionals, indented one level deeper each time:

```python
age = 20
has_ticket = True

if age >= 18:
    if has_ticket:
        print("Welcome in.")
    else:
        print("You need a ticket.")
else:
    print("You must be 18 or older.")
```

**Output:**

```
Welcome in.
```

Nesting is sometimes necessary, but deep nesting (3+ levels) usually signals the logic could be flattened using `and`/`or` or restructured with early returns (more relevant once you're writing functions). Keep an eye on this as your conditionals grow more complex later in the sheet.

### 9. Truthy and Falsy Values — A Core Python Concept

This is a genuinely important Python-specific idea: **you don't always need an explicit comparison operator in an `if` condition.** Python will evaluate _any_ value for its inherent "truthiness."

Values that are automatically treated as **False** in a condition:

- `0` (and `0.0`)
- `""` (empty string)
- `[]` (empty list)
- `{}` (empty dictionary)
- `None`

Everything else is treated as **True**, including non-zero numbers, non-empty strings, and non-empty lists.

```python
arr = []

if arr:
    print("List has elements.")
else:
    print("List is empty.")
```

**Output:**

```
List is empty.
```

This is extremely common in real DSA code — checking `if arr:` instead of `if len(arr) > 0:` is considered the more idiomatic, "Pythonic" way to check for a non-empty list. You'll see and write both, but recognize `if arr:` when you encounter it — it's not a shortcut hiding a bug, it's standard practice.

---

### 10. Ternary (Conditional) Expressions

For simple two-way branches that just assign a value, Python offers a compact one-line form instead of a full `if-else` block:

```python
age = 20
status = "adult" if age >= 18 else "minor"
print(status)
```

**Output:**

```
adult
```

**General form:** `value_if_true if condition else value_if_false`

This is purely a convenience for short, simple assignments — don't force complex logic into one line just to use it. If the condition or the branches get complicated, a regular `if-else` block is more readable and that readability matters more than compactness.

## Cheat Sheet — 1.3 If-Else

|Task|Code|
|---|---|
|Basic if|`if condition:`|
|If-else|`if condition: ... else: ...`|
|Multiple branches|`if ...: elif ...: elif ...: else: ...`|
|AND|`and`|
|OR|`or`|
|NOT|`not`|
|Chained comparison|`25 <= x <= 44`|
|Check non-empty list|`if arr:`|
|Check empty list|`if not arr:`|
|One-line conditional assignment|`x = a if condition else b`|

## Common Pitfalls to Avoid

- **Forgetting the colon (`:`)** at the end of `if`, `elif`, or `else` lines — this is a syntax error every single time.
- **Inconsistent indentation.** Mixing tabs and spaces, or indenting inconsistently within the same block, will cause an `IndentationError`. Stick to 4 spaces, always.
- **Using `=` instead of `==`.** `=` is assignment, `==` is comparison. Writing `if age = 18:` is a syntax error in Python (unlike some languages where it silently does the wrong thing) — Python will actually stop you here, which is a small mercy.
- **Writing `else if` instead of `elif`.** Python will not recognize `else if` as valid — it must be the single word `elif`.
- **Relying on truthy/falsy checks without meaning to.** `if x:` behaves very differently depending on whether `x` is `0`, `None`, an empty list, or something else — make sure that's actually the check you intend, especially with numeric values where `0` being falsy can silently skip logic you expected to run.
